git add - позволяет добавить файлы в stage, stage - это промежуточное состояние перед отправкой на git, то есть мы фиксируем изменения
git reset - убирает из stage
git commit - зафиксировать наши изменения, то есть создать какую-то ссылку, чтобы git понимал, что наши изменения будут храниться в коммите
ветка - это папка, в которой есть подпапки(коммиты), можно создавать множество веток
git branch -M main
git commit -m "first commit"
git log - посмотреть список коммитов
git remote - к локальному репозиторию подключаем удаленный
git remote -v - посмотреть какие репозитории подключены
git push - отправить данные на удаленный репозиторий
git status - есть ли в stage файл
git clone - с удаленного репозитория в локальный
